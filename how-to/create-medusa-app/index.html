Using create-adminfirstly-app

With the new create-adminfirstly-app tool you will get your Adminfirstly development environment ready within a couple of minutes. After completion, you will have a Adminfirstly backend, a Gatsby or Next.js storefront, and an admin dashboard up and running on your local machine.

Starting a new e-commerce project just got easier, now with one command.

Getting started with create-adminfirstly-app​

Use create-adminfirstly-app with npx:

npx create-adminfirstly-app

Or Yarn:

yarn create adminfirstly-app

Behind the scenes, create-adminfirstly-app is populating your database with some initial set of mock data, which helps to interact with Adminfirstly setup intuitively straight away.

Right after hitting one of those commands, the multistep installation process will be initiated, so the starter can be shaped right for the specific needs.

Destination folder​

Enter the path to the directory that will become the root of your Adminfirstly project:

? Where should your project be installed? › my-adminfirstly-store

Pick the starter you prefer​

? Which Adminfirstly starter would you like to install? …
❯ adminfirstly-starter-default
  adminfirstly-starter-contentful
  Other

You will be presented with three options:

  • adminfirstly-starter-default is the most lightweight version of a Adminfirstly project

  • adminfirstly-starter-contentful almost like the default starter, but with adminfirstly-plugin-contentful preinstalled

  • Other if you have a different starter that you would wish to install from Other will give you the option of providing a URL to that starter. An additional question will be asked if you choose this option:

    Where is the starter located? (URL or path) › https://github.com/somecoolusername/my-custom-adminfirstly-starter

For the walkthrough purposes, we assume that the selected starter is adminfirstly-starter-default and proceed to the next step.

Selecting a Storefront​

After selecting your Adminfirstly starter you will be given the option to install one of our storefront starters. At the moment we have starters for Gatsby and Next.js:

Which storefront starter would you like to install? …
❯ Gatsby Starter
  Next.js Starter
  None

You may also select None if the choice is to craft a custom storefront for your product.

create-adminfirstly-app now has all of the info necessary for the installation to begin.

Creating new project from git: https://github.com/adminfirstly/adminfirstly-starter-default.git
✔ Created starter directory layout
Installing packages...

Once the installation has been completed you will have a Adminfirstly backend, a demo storefront, and an admin dashboard.

What's inside​

Inside the root folder which was specified at the beginning of the installation process the following structure could be found:

/my-adminfirstly-store
  /storefront // Adminfirstly storefront starter
  /backend // Adminfirstly starter as a backend option 
  /admin // Adminfirstly admin panel

create-adminfirstly-app prints out the commands that are available to you after installation. When each project is started you can visit your storefront, complete the order, and view the order in Adminfirstly admin.

⠴ Installing packages...
✔ Packages installed
Initialising git in my-adminfirstly-store/admin
Create initial git commit in my-adminfirstly-store/admin

  Your project is ready 🚀. The available commands are:

    Adminfirstly API
    cd my-adminfirstly-store/backend
    yarn start

    Admin
    cd my-adminfirstly-store/admin
    yarn start

    Storefront
    cd my-adminfirstly-store/storefront
    yarn start
