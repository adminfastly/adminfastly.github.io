CORS issues

If you are experiencing connection issues when trying to access your Adminfirstly instance from a storefront, it is most likely due to CORS issues.

You might see a log in your browser console, that looks like this:

In your adminfirstly-config.js , you should ensure that you've configured your CORS settings correctly. By default, the Adminfirstly starter run on port 9000, Adminfirstly Admin on port 7000, and the storefront starters on port 8000.

The default configuration uses the following CORS settings:

// CORS when consuming Adminfirstly from admin
const ADMIN_CORS = process.env.ADMIN_CORS || "http://localhost:7000"

// CORS to avoid issues when consuming Adminfirstly from a client
const STORE_CORS = process.env.STORE_CORS || "http://localhost:8000"

If you wish to run your storefront or Adminfirstly admin on other ports, you should update the above settings accordinly.
